Lighthouse color system v0.1

从灯笼的暖光，转向灯塔的信号系统。

Lighthouse 的视觉底层不再以米黄、固定暖色和纸张感为主。新的色彩系统要更清晰、更稳定、更有导航感，同时允许每套方案拥有不同光质。

色彩原则 核心色板 语义与状态 产品预览 界面样例 Token 草案

Design judgment

Lantern 是近身照明，适合柔和、暖、手持感。Lighthouse 是远距导航，适合高对比、清晰边界、稳定方向和少量强信号色。

  • 大面积背景从米黄改为雾白和浅石灰，不让页面发旧。
  • 信号色不固定为琥珀；它可以是暖光、冷光或金属标记。
  • 主色改为每套方案各自的导航色，承载秩序、信任和系统感。

01 Principles

色彩不是装饰，而是导航。

这套颜色要服务三件事：让一线员工看得清，让管理者判断得快，让平台从“阅读材料”转向“服务文化操作台”。

信号稀缺 Signal Light 只给当前状态、关键行动和需要注意的判断点，不再铺满页面。
背景退后 背景用雾白、石灰和纸白，不抢内容，让知识和操作成为第一视觉层级。
边界明确 减少半透明和 blur，容器边界更实，适合工作场景中的长期阅读。
温度保留 保留少量暖色和柔和中性色，避免系统变成冷冰冰的后台。

02 Core palette

核心色板：导航、雾面背景、信号和深墨。

主色不再由暖色承担，而是由每套方案的主导航色负责系统秩序；Signal Light 退到强调层，只处理定位、提醒和关键行动。

Deep Ink #0F2D44

主要文字、深色导航、强对比背景。比纯黑更柔和，但足够稳。

Fog Paper #F2F8F8

主背景。保留一点暖度，但比旧米黄色更轻、更干净。

Primary Navigation #176E8B

新的主导航色。用于导航、主要按钮、选中状态和信息结构。

Signal Light #28B8C7

灯塔信号色。不限定为琥珀，只给高价值提醒、核心 CTA 和当前定位。

Evidence / Info #3E89A8

信息、解释、搜索、Hermit 引用依据。避免科技蓝的廉价感。

Practice / Success #5D9F86

通过、完成、可复用经验。适合服务文化中的正向反馈。

Risk / Boundary #C65E52

驳回、风险、禁止行为。比鲜红更克制，适合企业内部语境。

Stone Line #BFD8DE

边框和分隔线。比白色边框更清楚，也比灰色更贴合品牌。

03 Neutral scale

中性色：不再发灰，而是带轻微品牌温度。

旧界面的问题不是“颜色少”，而是大量低透明文字叠在浅米色背景上，造成阅读发虚。新系统直接定义清晰层级，少依赖透明度。

04 Contrast pairs

可读性组合先于审美组合。

工作场景中的正文、说明、表格和审核状态必须清楚。低对比可以用于禁用、辅助和装饰，但不再承担正文阅读。

Ink on Chalk 主要正文、标题、表格内容。默认阅读组合。
Chalk on Ink 侧边导航、重要容器、聚焦场景。高稳定感。
Chalk on Coast 主要按钮、激活导航、关键系统动作。
Text on Signal 信号按钮和短标签。不同方案可以使用不同光质，避免长段正文使用。
Ink Soft on Fog 说明性内容、空状态、次级提示。
Chalk on Coral 风险和禁止提示。只在需要强提醒时使用。

07 Usage rules

使用规则：少量颜色，明确职责。

真正的变化不是“从黄色换成蓝绿色”，而是让每个颜色在产品中有固定职责。

Do

  • 主导航和主要行动使用 Primary Navigation。
  • Signal Light 只用于当前定位、关键 CTA 和高价值提醒。
  • 正文优先使用 Deep Ink 或 Ink Soft，避免透明度正文。
  • 状态色必须和文字、图标、标签一起使用，不只依赖颜色。

Don't

  • 不要把信号色大面积铺开，否则会重新变成单一色调。
  • 不要用低透明灰字承担正文阅读。
  • 不要用半透明白卡和 blur 承载核心工作区。
  • 不要把所有模块都染成同一种主色。
