Foundation

视觉底层先收住

当前阶段把 A 配色当作默认方向，但真正要解决的是可读性、层级、密度、组件一致性和状态表达。Kit 定稿后再迁移到真实页面，避免边改页面边改规范。

Default: A / Harbor Signal
Visual Language

海图、信号、控制台

质感不靠大面积琥珀或装饰卡片，而是来自可解释的材料：细密坐标线、深海蓝控制面、清亮信号色、少量黄铜标记。它应该像一个服务文化导航系统，而不是一份被美化的文档。

设计原则

  • 1 企业服务工具优先界面要稳定、清楚、可反复使用，不走文艺阅读稿或活动页方向。
  • 2 清晰层级优先正文、弱文本、状态标签和操作按钮都要有明确职责，避免浅、散、糊。
  • 3 先 Kit，后替换组件形态、页面结构和状态语言定稿后，再替换原版实现。

核心 Token

  • 页面底色--ui-page 用于应用背景，避免纸张旧感。
  • 承载面--ui-panel 用于卡片、表格、聊天面板。
  • 主操作--ui-primary 用于主按钮、当前导航、关键强调。
  • 信号色--ui-signal 用于提醒、AI 协助、选中辅助，不再是全局琥珀。
  • 圆角业务组件默认 8px 以内，工具界面不做大圆角卡片感。

Grid System

网格系统先定骨架

网格不是背景装饰，而是页面迁移时的结构合同。Lighthouse 采用 4pt 间距刻度、12 栏内容画布和组件级自适应栅格，让 Heart、Workshop、Hermit 的页面密度有统一依据。

布局规则

  • 页面画布max-width: 1220px，桌面端左右安全边距 28px，移动端 16px。
  • 主栅格桌面采用 12 栏，内容区可按 3/9、4/8、6/6、4/4/4 组合。
  • 组件栅格卡片内部优先用 4pt 间距刻度和局部二栏，不在小组件里继续套复杂栏。
  • 纵向节奏Section 间距 22-28px，卡片内间距 12-18px，表格行高保持稳定。
  • 触控下限按钮、图标按钮和可点标签的触控区域不低于 42px，移动端优先 44px。

4pt 间距刻度

用 4pt 做底层刻度，避免 8pt 系统在中文工具界面里过粗。常用间距保持在 8、12、16、24、32，48 以上只用于大区块分隔。

Typography

字体换成不锋利的非衬线

产品界面不再使用宋体感或文学稿件气质。标题、正文、按钮统一使用柔和非衬线，通过字重、字号、间距和色值建立层级。

No Serif UI

推荐栈：Aptos、Noto Sans SC、Microsoft YaHei UI、Microsoft YaHei、PingFang SC、Source Han Sans SC。不使用负字距，不用随视口缩放字号。

使用规则

  • 导航20px，优先 -bold，当前页使用主色底。
  • 工具按钮18px，线性或实心都可，但同一工具栏内保持一致。
  • 状态图标跟随状态色，不单独引入新的颜色。
  • 禁用项只降低对比度和边框，不换成另一套图标表达。
  • 禁止不混用 Lucide、Heroicons、Material Symbols 等其他家族。

Redesign Contract

改造不是一一对应换皮

真实前端本身在信息层级、可读性和任务密度上需要重做。迁移时保留该有的业务内容和用户任务，但允许重排首屏、拆分长文、合并弱入口、删掉旧视觉噪音。

Readability First

迁移判断顺序：先确认页面职责和核心用户任务，再确定内容优先级、阅读路径、组件密度和状态入口。旧页面结构只作为材料来源，不作为新页面布局的约束。

保留内容责任

保留用户需要理解、判断和行动的信息，不保留旧页面的装饰层、低对比层和重复说明。

Content
  • 每页先列必须回答的问题。
  • 相同含义的标题、说明和标签合并。
  • 引用、来源、状态说明放到固定位置。

重建信息层级

首屏先给判断入口，正文按任务分块，弱辅助信息进入 meta、侧栏或折叠区。

Hierarchy
  • 标题、摘要、主操作不被长段落淹没。
  • 卡片内部使用固定轨道和 4pt 间距。
  • 桌面优先横向比较，移动端优先单列阅读。

提高可读性

减少 serif 正文、低透明文字和大块松散留白，转为更稳的工具型阅读密度。

Clarity
  • 普通正文不低于 14px，关键说明保持高对比。
  • 长文拆成摘要、要点、证据和行动。
  • 状态色只表达状态，不承担装饰。

Migration Contract

定稿后替换原版的规则

这不是只换几组颜色变量，也不是照旧页面逐块复刻。定稿后要按层替换：基础 token、字体、控件、页面结构、业务组件、状态语言；页面层允许围绕信息层级和可读性重新组织。

Do Not Patch Randomly

需要替换掉的旧特征

  • #F3EFE0、浅米纸张底色容易把工具界面带回文艺稿件感。
  • text-ink/50、text-ink/30正文和辅助信息可读性不足。
  • serif / 宋体感正文只保留在内容材料或引用场景，不作为 UI 默认字体。
  • 大面积琥珀强调改为主色 + 信号色 + 状态色分工。
  • 玻璃卡片、过大圆角、松散留白改为稳定面板、8px 圆角和明确网格。

替换顺序

  • 1Token and type先替换全局色彩、字体、字号、边框、阴影、圆角。
  • 2Primitive controls按钮、输入框、标签、分段控件、状态块统一。
  • 3App shell导航、顶部搜索、内容容器和页面操作区统一。
  • 4Page redesign pass按页面职责重排首屏、阅读路径、信息密度和操作优先级。
  • 5Product components再替换 Heart、Workshop、Hermit 的业务组件。

Lighthouse UI Kit v0.1 defines the visual, component, and redesign contract. It does not change the production React implementation yet.
