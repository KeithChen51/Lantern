Lighthouse Amber Visual System

同一束光，在不同页面保持同一种秩序。

新版视觉系统不把首页、内容页和工作页拆成三套风格，而是用同一组灯塔母题、琥珀色、暖纸材质、宋体标题、苹方控件和克制动效，形成从品牌精神到日常操作的连续体验。

Core Decision

首页可以最有画面感，但它不是例外。它只是 Lighthouse Amber 在最高表达强度下的状态；镜鉴、笃行、路引和后台使用同一视觉 DNA 的低强度版本。

01 / Thesis

不是品牌封面一套，产品页面一套。

系统的目标是避免割裂。所有页面共享同一视觉根系，只根据任务强度调节影像、材质、字号、密度和动效。

Mother Motif 灯塔、光、雾、纸、琥珀、秩序

所有页面都从这六个母题出发。首页表现“远方与光”，内容页表现“纸面与阅读”，工作页表现“方向与判断”。

Shared Components 同一导航、同一字体系、同一组件逻辑

导航不是页面装饰，而是系统框架。搜索、消息、焦点、按钮、标签、表格都继承同一套参数。

Governed Intensity 变化的是强度，不是风格

Immersive、Editorial、Operational 是一个滑杆上的三个档位，不是三个主题包。

02 / Expression Spectrum

三种表达强度，共用一套系统。

首页首屏可以有影像和雾光；价值正文回到暖纸和阅读节奏；路引、Workshop、后台进入更密集的工作界面。它们的差别应该像同一种材料的不同加工方式。

HighestImmersive

不能扩散到表格、后台、普通卡片和所有弹层。

03 / Color Tokens

颜色不是主题切换，而是场景分工。

深夜墨色负责灯塔与沉浸，暖纸色负责阅读，琥珀负责方向和激活，黄铜负责辅助状态。五个价值词保留小标签映射，不成为页面主色。

深夜墨色--lh-ink-night
暖纸底色--lh-paper
琥珀光色--lh-amber
黄铜辅助--lh-brass-text

04 / Material System

玻璃、雾光和纸面要有明确职责。

旧规范担心玻璃变成装饰，这是对的。新版不是放开使用，而是规定玻璃只属于浮动导航、搜索、消息、封面控制和少量沉浸叠层。

本心镜鉴笃行路引

影像和雾光只用于建立方向感，不代替内容秩序。

允许
  • 首页首屏和关键封面使用影像、暗色雾面和远处灯塔。
  • 浮动导航、搜索、消息和临时工具使用半透墨色玻璃。
  • 内容阅读和工作区回到暖纸、白面、细边框和低阴影。
禁止
  • 把所有卡片、表格、表单都做成玻璃。
  • 用低透明度文字制造高级感，导致可读性下降。
  • 给每个模块单独换背景、换主题色、换导航样式。

05 / Typography

文化感要从字体系贯穿，而不是只停在首页。

思源宋承担价值表达和阅读标题，苹方承担操作、导航、正文和数据。首页只是字号最大，其他页面仍然保留同一语气。

07 / Page Archetypes

三个页面示例必须像同一个系统。

下面不是三个视觉方向，而是同一系统在不同任务里的降噪结果。导航、字体、颜色、边框、半径和焦点规则保持一致。

Immersive 首页

首屏承载品牌精神和价值主句。影像负责氛围，内容仍左对齐、留白清晰、导航稳定。

Editorial 叙事

本心正文、通知内容、镜鉴和笃行长文使用纸面、宋体标题、稳定行高和少量琥珀标注。

Operational 工作

路引、Workshop、后台不做海报化，但保留同一导航、琥珀激活、暖纸底和清晰焦点。

08 / Component Rules

组件负责把品牌气质落到日常操作。

如果只有首页好看，系统仍然失败。按钮、输入、标签、表格和消息气泡必须继承同一套琥珀、纸面、半径、焦点与状态逻辑。

Buttons

主按钮使用琥珀底，次按钮使用纸面和细边框。loading 保持宽度，disabled 不只靠透明度。

确认发布保存草稿

Fields

输入控件保持暖纸或白面底，focus 使用 2px 琥珀 outline + offset + halo，不用只靠柔光。

Chips

价值词颜色只用于小标签、筛选和来源标记。不要让五个价值词变成五套页面主题。

求真尽善致美大爱

Tables

Operational 页面用数据密度建立信任。表头、状态、批量操作和空状态都要有清晰层级。

09 / Motion And Access

动效服务阅读顺序和状态反馈。

首页可以有一次性分层进入，但系统不能靠循环动画制造高级感。所有动画必须可关闭、可理解，并且不阻挡内容。

Entrance420-700ms / ease-out

首页主句、叙事 section、重要空状态可一次性进入，表现阅读顺序。

Interaction160-240ms / transform + opacity

hover、pressed、selected、popover 使用轻微位移、透明度和颜色变化。

Accessibilityprefers-reduced-motion

尊重 reduced motion；焦点、错误、加载不能只用动画表达。

10 / Migration Contract

后续迁移按系统强度，而不是按页面喜好。

每个页面先判断任务：建立精神、沉淀内容，还是完成工作。然后选择对应强度，并继承同一套导航、token、字体、组件和动效。

首页 / 本心允许 Immersive 首屏，但第二屏之后进入 Editorial；不能把所有内容做成封面。
镜鉴 / 笃行以 Editorial 为主，案例详情和复盘段落用长文排版，状态和入口使用 Operational 组件。
路引 / Workshop / 后台以 Operational 为主，但保持琥珀激活、暖纸底、宋体关键标题和同一导航。
全局规则新样式必须先进入视觉系统，再进入 runtime token 和 primitive；禁止页面局部发明第二套 UI。
