# 经销商年会前 H5 预热方案（形式稿）

> 当前文档保留 H5 互动形式、体验结构和结果页方向。旧版题目不作为后续定稿依据；手机端 demo 已基于“诚信经营”参悟案例形成首版 5 题，题目口径面向经销商门店现场，仍可继续审稿优化。静态 demo 已拆成独立手机端预览文件：`docs/design/dealer-service-h5-preheat-demo.html`。

## 项目定位

H5 不是知识答题，也不是单向介绍理念。它的定位是一次轻量的“服务优先级画像”：让经销商在真实售后场景中选择自己更看重的处理重点，提前感受到这次服务理念升级讲的是日常判断、客户信任和门店经营，而不是抽象口号。

## 核心概念

**一次服务里的五个判断**

用户在若干个服务判断场景中选择自己更优先关注的处理重点。每个选项都应成立，不设置明显错误答案；系统不评判对错，而是反馈每个优先级背后的服务理念，并在最后生成个人服务关键词。

推荐主线：

- 求真：把事实、依据、进度和不确定性讲清楚。
- 尽善：在规则、成本、效率和客户利益之间做更稳妥的取舍。
- 致美：把流程、秩序、专业和体面真正交付出来。
- 大爱：在规则之内看见人的处境、尊严、安全和长期关系。
- 幸福：用客户是否安心、员工是否有尊严、组织是否可持续来检验结果。

## 目标人群

经销商管理者、售后负责人、服务经理、服务顾问及与售后体验相关的门店团队。

## 体验流程

1. 开场页
   用一句问题进入服务现场：“一位客户进店后，服务从哪一刻开始？”强调服务不只发生在工单里，也发生在解释、等待、协调和补救中。

2. 场景选择
   保留“案例情境 + 优先级选项 + 即时反馈”的互动机制。首版题目围绕养护品推荐中的客户信任、团队执行、透明说明和长期复购展开，避免把年会预热做成制度讨论题、售后培训题或标准答案题。

3. 理念反馈
   反馈不说“你选对了”，也不暗示其他选项是错的，而是说“你更优先看重某种服务判断”。重点是把理念放回具体动作。

4. 结果画像
   根据用户选择生成“精诚服务关键词”，例如“求真型”“致美型”“求真 x 大爱型”。结果页直接生成一张完整竖版海报图，页面本身就是可保存、可分享的结果。底部保留两个动作：“保存服务关键词海报”和“查看服务理念升级白皮书”。

5. 分享预热
   自动生成的海报用于年会前转发、内部群预热或门店报名确认。白皮书入口用于承接进一步理解，但不把用户带入考试、积分或排名机制。

## 内容原则

- 不做考试感：避免分数、排行榜、标准答案。
- 不设错误选项：每个选项都代表一种可成立的服务优先级，只体现侧重点差异。
- 不做泛口号：每个理念必须对应一个服务场景和一个动作。
- 不做过度游戏化：经销商年会受众更适合稳重、专业、有参与感的互动。
- 不只面向客户：结果文案中保留员工支持和组织可持续的视角。
- 题目继续审稿：现阶段先验证形式、动效、结果页和分享链路，题目内容可继续基于参悟案例细化。

## 交付建议

首版静态 demo 先作为手机端预览使用，只验证流程、视觉风格、结果页形式和保存动作。题目内容确认后，再进入正式开发，补充真实登录、门店信息、海报下载、分享链路和埋点。
