Service Brand Visual Identity

精诚服务新版 VI 手册

将“求真、尽善、致美、大爱、幸福”转化为可识别、可执行、可复制的服务视觉系统。

核心概念：信任路径 Trust Path 设计原则：事实清楚、流程有序、专业可见、关系有温度 标识策略：官方标识保留，视觉系统更新

A0 / Manual Principles

手册使用原则

本手册是新版 VI 的工作稿，用于统一视觉语言、应用边界和交付判断。正式发布前，商标、字体授权和标准色值仍需品牌方确认。

01. 标识不重绘

官方标识、理念 icon 和联合标识规则属于正式品牌资产。除品牌方正式审批外，不重新绘制、不改色、不变形、不替换。

02. 系统要可执行

新版 VI 不只用于海报展示，更要进入客户解释卡、车间看板、员工支持卡、共创内容和数字平台。

03. 红色降级使用

精诚红保留为历史识别和重要提醒色。大面积背景与结构色优先使用信任深蓝、银灰和事实白。

04. 图形承载价值

“信任路径”承担新版价值表达：定点、转向、对齐、承托、闭环，对应求真、尽善、致美、大爱、幸福。

05. 文字先清楚

标题可以有东方气质，正文必须清楚稳定。服务说明、流程、风险和责任信息不得为了视觉效果牺牲可读性。

06. 真实优先

图像应来自真实服务现场、真实工序、真实人物互动。避免过度摆拍、炫技光效和无法兑现的情绪化表达。

A1 / Brand Marks

品牌标识规范

第一阶段沿用旧版手册中的正式标识资产，同时新增“求真、尽善、致美、大爱、幸福”十字核心理念标识。它预计取代“精于勤，诚于心”的日常理念表达。

比亚迪精诚服务横版标识

  • 用于正式对外传播、门店物料、汇报封面、品牌落款。
  • 保持原始比例、颜色和留白，不拆分印章、理念和英文签名。
  • 在复杂背景上必须使用经确认的正色或反白版本。

十字核心理念标准稿

  • 作为新版价值框架的主理念表达，优先用于品牌升级、培训、手册封面和核心传播页。
  • 五组二字词保持完整顺序，不拆分重排，不单独抽取某一词作为口号。
  • 默认横排标准稿；小尺寸使用紧凑稿；章节页可使用竖排稿。

腾势精诚服务横版标识

  • 用于腾势相关售后服务传播和内部材料。
  • 与主品牌标识共同出现时，遵守旧版手册中的联合标识规则。
  • 不得借新版 VI 改变腾势蓝或组合关系。

旧版理念 icon（历史资产）

  • 保留用于品牌历史、文化溯源、老物料过渡和纪念内容。
  • 新版日常理念表达逐步转向“求真、尽善、致美、大爱、幸福”。
  • 不得将旧理念 icon 与十字核心理念并列为同级主口号。

Combination Rules

主品牌与核心理念共同出现时，主品牌优先，精诚服务承接，核心理念解释。

组合层级

主品牌标识优先，精诚服务标识次之，十字核心理念作为价值解释层。三者不得等大并列。

使用场景

完整组合用于封面、主视觉、发布页和大型物料；小尺寸场景只保留主品牌 + 精诚服务标识，不强行加入十字理念。

安全距离

主品牌与精诚服务之间使用垂直分隔线或足够留白；核心理念与标识之间至少保留标识高度 0.5A。

  • 安全空间标识四周保留不小于标识高度 0.5A 的空白；联合标识可按 0.25A 作为最低参考。
  • 最小尺寸横版标识高度小于 10mm 时禁止使用；竖版标识高度小于 16mm 时禁止使用。
  • 背景选择浅色背景使用正色稿；深色背景使用反白稿或经确认的深色版。
  • 文件来源优先使用官方 AI/PNG 文件，不建议根据网格图自行重绘。

A4 / Graphic System

信任路径辅助图形

辅助图形由线、节点、转向、对齐、承托和闭环组成。它是新版价值框架的可视化系统，不是替代标志的第二套 logo。

定点事实、证据、依据、责任人的标记。
转向方案选择、协调路径、规则中的更优解。
对齐工序、现场、标准和专业交付的秩序。
承托客户处境、员工支持、安全边界和尊严。
闭环安心、托付、复盘、持续改善和长期关系。

A5 / Layout System

版式与信息层级

新版版式围绕“事实层、判断层、关系层”组织。信息先清楚，再强调品牌气质。

事实层

用于车况、费用、进度、依据、责任人。白底、深蓝标题、灰色正文，避免装饰干扰判断。

判断层

用于方案比较、路径转向、边界说明。服务蓝作为结构强调，暖金只标记关键取舍。

关系层

用于客户安心、员工支持、长期托付。留白更大，文字更克制，用承托面表达温度。

A6 / Image Style

图像风格规范

图像要服务“可信”。优先真实服务现场、真实工序、真实人物互动，不使用无法支撑服务承诺的空泛情绪画面。

推荐方向

  • 服务顾问说明车况、费用、进度和风险。
  • 维修、钣喷、质检、交付的真实工序。
  • 工具、工单、检查表、洁净工位等细节特写。
  • 员工协作、培训、复盘和现场支持。

避免方向

  • 过度摆拍的握手、递钥匙、微笑合影。
  • 只有车辆大景，缺少服务关系和服务动作。
  • 过度暗调、强反光、泛科技光效。
  • 暗示“绝对安全”“零风险”等不可兑现承诺。

C1 / Misuse

禁用示例

以下规则用于保护正式标识和新版视觉系统的一致性。真实项目中应由模板和校验机制进一步约束。

禁止拉伸变形

不得横向或纵向压缩、拉伸、倾斜标识。

禁止随意改色

不得把官方标识改成非手册规定的颜色组合。

禁止复杂背景

不得把标识直接放在高噪声、强纹理或低对比背景上。

全红背景
禁止红色过载

新版 VI 不以大面积红色作为默认主视觉。

五色贴纸化
禁止普通 icon 化

五个价值维度不能简化成随意五色图标贴纸。

无意义装饰线
禁止装饰性路径

信任路径必须表达流程、判断或关系，不做纯背景花纹。

D1 / Delivery Checklist

落地交付清单

正式进入模板和物料制作前，建议先用这份清单做一次品牌、法务、授权和应用场景确认。

  1. 确认是否继续采用“The Genuine Way”作为新版英文表达。
  2. 确认“求真、尽善、致美、大爱、幸福”正式替代“精于勤，诚于心”的日常理念表达。
  3. 确认十字核心理念标准稿、紧凑稿、竖排稿三种字样是否进入正式规范。
  4. 确认旧标识、历史理念 icon、联合标识的正式文件版本。
  5. 确认精诚红标准色：旧稿中的 RGB 229/0/17 与 HEX #ff0000 需要统一。
  6. 复核方正、汉仪、BYD Type 等字体的授权范围。
  7. 确认 BYD、腾势、仰望、方程豹等主品牌与核心理念共同出现时的组合层级。
  8. 补充真实服务现场照片，建立正向与反向图片库。
  9. 制作 PPT、客户解释卡、车间看板、员工支持卡、数字平台组件模板。
  10. 建立禁用示例和文件校验机制，避免输出漂移。
