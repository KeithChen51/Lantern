# 精诚服务新版 VI 方案（工作稿）

版本：v0.1
日期：2026-06-24
依据材料：

- 旧版《精诚服务品牌管理手册》PDF/PPTX，共 104 页。
- 旧版品牌资产目录：品牌标识、组合 logo 使用规范、字体包。
- 新版价值框架：《精诚服务品牌价值观纲领白皮书》《精诚服务 (The Genuine Way) 品牌价值观框架与落地指南》。

## 1. 设计命题

新版 VI 不应只是把旧版“精诚红、印章、书法理念”换一套更现代的皮肤，而要回答一个新的品牌问题：

**如何把“持续经营客户信任”做成可见、可用、可复制的视觉系统。**

旧版手册建立的是识别规范：标志怎么放、颜色怎么用、字体怎么选、联合标识如何组合。新版 VI 需要进一步承担“价值表达”和“组织执行”的功能：让客户看见事实、流程、专业和善意，也让员工在日常材料、培训内容、数字化系统和服务现场中感受到清晰的判断尺度。

新版核心表达建议为：

> **让信任被看见，让服务可托付。**

这句话不是替代品牌口号，而是新版 VI 的设计原则。它要求所有视觉元素都服务于三件事：事实清楚、过程有序、关系有温度。

## 2. 从旧系统继承什么

旧版手册中值得继承的，不是单一视觉风格，而是几条已经被长期使用验证过的资产和原则。

### 2.1 必须继承的资产

- **“精诚服务”品牌名与历史识别资产**：旧标志、印章、书法理念 icon 是品牌资产，不应在没有正式审批的情况下重绘、变形、改色或重新解释。
- **“精于勤，诚于心”的历史表达**：它仍然是“精诚”的根，但新版不再只停留在这句理念上，而是展开为求真、尽善、致美、大爱、幸福的价值路径。
- **官方品牌标识组合规则**：BYD、腾势、仰望等品牌标识与精诚服务标识的组合，应继续遵守对角、留白、正反色、最小尺寸等基本规则。
- **字体资产**：旧版已包含方正清刻本悦宋简体、方正兰亭系列、BYD Type 等字体资产。新版应先复用这些已采购/授权字体，再补充数字端备用字体。

### 2.2 必须继承的表达原则

旧版沟通调性提出“真实的、开放的、清晰的、有温度的”。这四个原则与新版价值框架高度一致，可以保留并升级：

- 真实的 -> **求真**：事实、依据、进度、风险讲清楚。
- 清晰的 -> **致美**：流程、秩序、责任和交付可被看见。
- 有温度的 -> **大爱**：看见客户和员工的具体处境。
- 开放的 -> **尽善**：在规则和现实约束中寻找更好的解法。

## 3. 需要调整的旧系统问题

旧系统适合“识别管理”，但不足以支撑新版价值框架。

1. **红色承担过多识别压力**
   旧版比亚迪精诚服务以红色印章和红/灰/银为主。红色有历史识别度，但如果继续大面积使用，容易把服务表达推向“强提醒、强宣传”，不利于表达信任、秩序、专业和安心。

2. **理念 icon 过于依赖书法情绪**
   “精于勤，诚于心”的书法 icon 有文化感，但不适合承载所有数字化、培训、流程和服务场景。新版需要一套更模块化、更可解释的价值图形系统。

3. **不同品牌段落视觉差异较大**
   BYD 段落偏红灰银，腾势段落偏蓝黑银，仰望段落偏黑白银灰加工艺质感。新版如果要服务“精诚服务”整体升级，需要建立一个跨品牌一致的母系统，再允许不同品牌在局部应用中适配。

4. **应用场景偏办公物料，缺少数字化和服务现场语言**
   旧版重点覆盖 PPT、函件、名片、旗帜、胸针等。新版需要覆盖数字平台、服务流程图、客户解释卡、一线共创内容、Do/Don't 指南、培训材料和服务现场触点。

## 4. 新版 VI 总体方向

### 4.1 核心视觉概念：信任路径

新版 VI 建议以 **“信任路径 Trust Path”** 作为主视觉概念。

信任不是一个静态符号，而是服务过程中逐步建立的结果。视觉系统应从“一个 logo”扩展为“一条路径”：客户从问题进入服务场景，经过事实确认、方案选择、专业交付、关系照顾，最终抵达安心和愿意继续托付。

五个价值维度对应五个视觉动作：

| 价值维度 | 视觉动作 | 视觉含义 |
| --- | --- | --- |
| 求真 | 定点 | 把事实、依据、责任人标清楚 |
| 尽善 | 转向 | 在规则和现实约束中选择更好的解法 |
| 致美 | 对齐 | 让流程、工序、现场和交付保持秩序 |
| 大爱 | 承托 | 看见人的处境，给出支持和保护 |
| 幸福 | 闭环 | 让客户安心、员工有尊严、组织可持续 |

这套“路径语言”可以转化为线条、节点、卡片、流程图、服务地图、海报构图和数字界面组件。

### 4.2 视觉气质

新版 VI 的气质建议为：

**清晰、可信、克制、有温度。**

它不是高调炫技，也不是传统公益式温情；它应该像一个可信赖的服务组织：讲事实、守秩序、懂专业，也看见人。

具体设计判断：

- 版式：网格清楚，信息层级明确，留白充足但不空。
- 色彩：以深蓝、银灰、白和暖色点缀建立信任感，红色退为历史资产和重点提示色。
- 字体：标题保留东方气质，正文和数字端强调可读性和执行效率。
- 图形：从印章式静态符号，升级为路径、节点、秩序线和承托面的动态系统。
- 图片：真实服务场景、真实工序、真实人物，不使用过度摆拍和情绪煽动。

## 5. 品牌标识策略

### 5.1 不建议立即重绘官方标志

旧版目录中已有 AI/PNG 标志文件，包括比亚迪精诚服务、腾势精诚服务、仰望精诚至臻等。第一阶段不建议直接重绘这些官方标志。原因是：

- 现有标识已经被旧手册规范化，属于正式品牌资产。
- 标识涉及商标、法律、主品牌关系和多品牌组合规则。
- 新版 VI 的核心问题不是“把 logo 画新”，而是让视觉系统能够承载新价值框架。

建议采用 **“标识保留，系统更新”** 的路线：保留官方标识，新增一套“信任路径”视觉系统，用于承载求真、尽善、致美、大爱、幸福。

### 5.2 标识分层

| 层级 | 用途 | 规则 |
| --- | --- | --- |
| 官方品牌标识 | 正式对外传播、门店、物料、合同、函件 | 使用旧手册/官方资产，不重绘、不改色、不变形 |
| 联合品牌标识 | BYD/腾势/仰望等主品牌共同出现时 | 沿用对角组合、留白、正反色和最小尺寸规则 |
| 历史理念标识 | 品牌历史、纪念内容、文化溯源 | “精于勤，诚于心”作为历史资产，不再承担所有新版价值表达 |
| 新版价值图形 | 培训、数字系统、海报、服务地图、共创内容 | 使用“信任路径”系统表达五个价值维度 |

## 6. 色彩系统

### 6.1 色彩策略

新版色彩不建议继续以“精诚红”作为大面积主视觉，而建议将红色作为历史识别和关键强调色，主色转向更稳定、更可信、更适合数字化和服务场景的深蓝/银灰系统。

建议色彩比例：

- 60%：白、雾白、浅银灰，用于背景和信息承载。
- 20%：信任深蓝，用于主标题、导航、关键结构。
- 10%：秩序灰/工艺银，用于边框、分隔、说明、辅助图形。
- 5%：服务蓝，用于进度、链接、流程节点。
- 5%：暖金/暖红，用于人文关怀、重点提醒、完成反馈。

### 6.2 建议色彩令牌

| 令牌 | 建议色值 | 来源/关系 | 用途 |
| --- | --- | --- | --- |
| 信任深蓝 Trust Ink | `#141B25` | 旧手册腾势段落“精诚蓝” | 主视觉底色、标题、数字系统主结构 |
| 服务蓝 Service Blue | `#003087` | 旧手册腾势蓝 | 流程节点、链接、服务路径强调 |
| 秩序灰 Order Gray | `#595757` | 旧手册精诚灰/陨石灰 | 正文标题、说明文字、边框 |
| 工艺银 Craft Silver | `#DCDDDD` / `#BABABA` | 旧手册精诚银、赫兹银 | 背景、分割、工艺质感、辅助图形 |
| 事实白 Fact White | `#FFFFFF` | 旧手册皓月白 | 大面积背景、信息清晰承载 |
| 善意金 Kindness Gold | `#BE9457` | 旧手册辅助金色 | 关怀、荣誉、员工尊严、重要但非警示信息 |
| 行动暖红 Action Red | `#E50011` | 由旧手册 R229/G0/B17 推导；旧稿也写有 `#ff0000` | 历史识别、重要提醒、品牌印章资产 |
| 结果黄 Outcome Yellow | `#FDD000` | 旧手册辅助黄色 | 幸福、完成、正向结果点缀，慎用 |

注意：旧手册中“精诚红”同时出现 R229/G0/B17 和 `#ff0000`，正式制版前应由品牌方确认最终标准色。数字端建议先以 `#E50011` 作为由 RGB 推导的校准稿，同时保留旧资产原色不改。

### 6.3 五个价值维度的颜色映射

| 价值维度 | 主色 | 辅色 | 使用建议 |
| --- | --- | --- | --- |
| 求真 | 信任深蓝 | 事实白 | 事实说明、证据、进度、责任人 |
| 尽善 | 服务蓝 | 工艺银 | 方案比较、下一步、协调路径 |
| 致美 | 秩序灰 | 工艺银 | 工序、标准、现场、流程图 |
| 大爱 | 善意金 | 行动暖红 | 关怀、支持、尊严、安全提醒 |
| 幸福 | 结果黄 | 事实白 | 完成反馈、长期关系、复购/托付表达 |

## 7. 字体系统

### 7.1 中文字体

| 场景 | 首选 | 备用 |
| --- | --- | --- |
| 品牌标题/大标题 | 方正清刻本悦宋简体 | 方正兰亭黑简体 |
| 正文/说明/数字系统 | 方正兰亭黑简体、方正兰亭纤黑简体 | 微软雅黑、Microsoft YaHei UI、思源黑体 |
| 服务现场短语/强调 | 方正清刻本悦宋简体 | 方正兰亭黑简体 |

建议保留方正清刻本悦宋简体的“端庄、有文化根基”的气质，但控制使用范围。它适合标题、价值短句和品牌叙事，不适合大量正文和数字系统。

### 7.2 英文字体与数字

| 场景 | 首选 | 说明 |
| --- | --- | --- |
| 英文品牌名、标题、编号 | BYD Type | 延续旧手册资产，适合品牌化表达 |
| 数据、表格、流程编号 | BYD Type Medium / Bold | 需要统一数字宽度时另行测试 |
| 系统备用 | Microsoft YaHei UI / Segoe UI | 仅用于内部系统或字体无法加载时 |

### 7.3 字体使用原则

- 不再用书法字体承担全部品牌表达；书法应作为文化资产，不作为数字系统的主字体。
- 标题可以有东方气质，正文必须清楚、稳定、易读。
- 对外物料优先使用授权字体；数字系统必须准备可部署的备用字体方案。
- 字体包和购买信息需要重新核对授权范围、授权期限、可嵌入/可商用范围。

## 8. 图形系统

### 8.1 主图形：信任路径线

信任路径线是新版 VI 的核心辅助图形。它由“线、节点、转向、闭环”组成，可以用于：

- 服务流程图。
- 客户旅程图。
- 品牌价值海报。
- 培训材料章节页。
- 数字平台模块标题。
- Do/Don't 指南卡片。

基础规则：

- 线条以直线和轻微圆角转折为主，避免过度装饰。
- 节点不使用普通圆点堆叠，应根据五个价值维度形成不同节点形态。
- 路径必须有起点、过程和结果，不做无意义的背景线条。
- 图形可以截取局部，但要保持方向感和秩序感。

### 8.2 五个价值图形

| 价值维度 | 建议图形 | 使用场景 |
| --- | --- | --- |
| 求真 | 定点/坐标/证据标记 | 事实说明、检查项、问题定位 |
| 尽善 | 分岔后再汇合的路径 | 方案选择、协调、复盘 |
| 致美 | 网格、对齐线、工序层 | 工艺、流程、现场标准 |
| 大爱 | 承托弧、保护面、留白包裹 | 客户关怀、员工支持、安全边界 |
| 幸福 | 闭环、稳定终点、轻量完成标记 | 结果反馈、长期关系、服务完成 |

### 8.3 不建议的图形方向

- 不使用泛红色光效、渐变球体、科技感粒子等泛化汽车视觉。
- 不把五个价值做成普通五色 icon 贴纸。
- 不使用过度书法化、传统纹样化的装饰来解释“精诚”。
- 不使用随机曲线、情绪化插画来表达“温度”。

## 9. 图像风格

新版图像风格建议坚持 **真实、克制、可感知**。

优先拍摄/选择：

- 服务顾问与客户沟通事实、费用、进度的场景。
- 车间工序、工具摆放、维修过程、钣喷修复前后对比。
- 员工协作、复盘、培训、班前会等组织能力场景。
- 客户等待区、交车、问题解决后的自然互动。
- 细节特写：工单、检查表、工具、洁净工位、交付确认。

避免：

- 过度摆拍的握手、微笑、递钥匙。
- 只有车辆大景、缺少服务关系的画面。
- 过度暗调、强反光、炫技光效。
- 与事实不符的“极致”“绝对”“零风险”等视觉暗示。

## 10. 版式与信息层级

新版版式要体现“信任路径”的逻辑，建议形成三层结构。

### 10.1 事实层

用于回答“发生了什么、依据是什么、谁负责、下一步是什么”。

视觉方式：

- 白底或浅银灰底。
- 深蓝标题。
- 灰色正文。
- 清楚的编号、时间、责任人和状态。

### 10.2 判断层

用于回答“为什么这么做、还有什么选择、如何在规则中尽善”。

视觉方式：

- 服务蓝节点。
- 路径分叉与合流。
- 方案对比表。
- 关键取舍使用暖金强调。

### 10.3 关系层

用于回答“客户是否安心、员工是否有尊严、组织是否可持续”。

视觉方式：

- 更大的留白。
- 真实人物或真实场景。
- 暖金/暖红小面积点缀。
- 结果闭环图形。

## 11. 应用系统建议

### 11.1 基础系统

- 标志与联合标识规范。
- 标志安全区、最小尺寸、正反色、禁用示例。
- 新版颜色系统与色彩比例。
- 中文/英文字体系统。
- 信任路径辅助图形。
- 五个价值维度图形。
- 图像风格规范。

### 11.2 服务现场

- 门店服务流程墙。
- 客户解释卡：事实、费用、进度、风险、下一步。
- 服务顾问桌牌/胸牌。
- 车间工序看板。
- 交车确认卡。
- 投诉/返修场景的安抚与处理指引。

### 11.3 内部培训与共创

- 求真/尽善/致美/大爱/幸福五类课程封面。
- Do/Don't 卡片模板。
- Workshop 共创提交模板。
- 案例复盘模板。
- 一线经验优秀案例海报。
- 员工支持与授权说明卡。

### 11.4 数字系统

适用于 Lighthouse / Hermit / Workshop 等数字化触点：

- 价值维度标签。
- 案例卡片。
- 知识文档封面。
- 证据链/引用/来源样式。
- 审核状态。
- 流程进度。
- 服务准则详情页。

数字端应避免把 VI 做成重装饰皮肤，而要把“清晰、可信、克制、有温度”落实到信息组织、状态提示、按钮层级和阅读体验。

### 11.5 对外传播

- 品牌升级发布主视觉。
- 经销商年会/服务大会物料。
- 公众号长图文。
- 海报与短视频片尾。
- 客户故事与服务案例。
- 服务承诺说明页。

## 12. 样张方向建议

第一批样张建议做 6 类，不急着做完整手册。

1. **品牌升级主视觉**
   展示“让信任被看见，让服务可托付”的主概念和信任路径图形。

2. **五个价值维度海报**
   每张对应一个维度，使用统一构图，但颜色和节点形态有差异。

3. **客户解释卡**
   验证“求真”和“尽善”是否能转成真实服务材料。

4. **车间工序看板**
   验证“致美”是否能从视觉美感落到流程、工艺和现场秩序。

5. **员工支持卡**
   验证“大爱”是否同时面向客户与员工，而不是单向要求一线。

6. **数字平台页面样张**
   验证 VI 在 Lighthouse / Workshop / Hermit 中是否可读、可用、可长期维护。

## 13. 新旧系统关系

新版 VI 与旧版手册的关系建议定义为：

| 旧版系统 | 新版处理 |

## 15. 当前待确认事项

1. **是否允许重新设计主标志**
   当前建议是不重绘官方标志，只新增价值图形系统。如果品牌方希望彻底换标，需要另起商标/法务/审批流程。

2. **新版英文名是否确认**
   现有新框架使用“精诚服务 (The Genuine Way)”。旧版标志中使用 “Superior and Sincere Services”。二者需要统一。

3. **红色的地位是否可以下调**
   如果内部仍要求红色为绝对主色，新版 VI 会更接近旧系统升级；如果允许红色退为强调色，新版会更适合“信任资产”和数字化服务场景。

4. **字体授权范围需要复核**
   字体目录中有授权字体包和购买信息，但正式对外/数字端嵌入前，需要确认授权期限、授权范围、Web/APP/视频/印刷等使用权限。

5. **是否覆盖全品牌矩阵**
   本稿按“精诚服务母系统”设计。如果要分别覆盖 BYD、腾势、仰望、方程豹，需要建立母系统 + 子品牌适配规则。

## 16. 一句话结论

新版 VI 的核心不是“换一套更好看的视觉”，而是把“求真、尽善、致美、大爱、幸福”变成一套可识别、可执行、可复制的服务视觉系统：事实清楚、流程有序、专业可见、关系有温度，最终让客户更安心，让员工更有尊严，让组织更可持续。
