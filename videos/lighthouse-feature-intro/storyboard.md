# Lighthouse 功能介绍视频脚本

时长：约 54 秒
画幅：1920 x 1080 横版
形式：中文屏幕文案，无旁白依赖
画面策略：不使用页面截图作为主要素材，而是把当前前端的 AppShell、Navigation、Header、页面标题、卡片、表单和问答区转写为 HyperFrames HTML/CSS 组件。

## 叙事结构

1. 0-6s：Lighthouse 定位
   面向汽车品牌售后体系的企业文化数字平台，把服务理念、标杆案例、内部实践、岗位规范和领域问答放在同一个工作台。

2. 6-14s：平台闭环
   展示 HTML 版 AppShell：侧栏导航、顶部搜索、身份入口和五个模块预览，说明这版视频展示的是前端结构本身。

3. 14-22s：Heart + Mirror
   转写 Heart 页面里的价值芯片、核心卡片，以及 Mirror 页面里的标杆案例卡片。

4. 22-30s：Action
   转写 Action 页面里的内部实践案例卡、标签和 Decision Nodes，突出案例不是结果展示，而是复盘选择。

5. 30-40s：Workshop
   转写 Workshop 的公共可见区、提交区、个人区域、审核区结构，重点展示 Do & Don't 表单和初审规则。

6. 40-48s：Hermit
   转写 Hermit 的欢迎态、建议问题、输入框和回答契约，保持“领域助手”定位。

7. 48-54s：收束
   学习、参照、实践、共创、问答形成闭环，让服务文化持续发生。

## 屏幕文案基调

正式、克制、面向企业内部汇报。避免夸张广告语，保留项目当前“学习和理解 + 实践沉淀 + 共创机制 + 领域问答”的阶段定位。
